1 BUSINESS REQUIREMENTS (Бізнес-вимоги)
1.1	Background (Передумови)

У сучасному світі туризм стає все більш популярним і доступним для широкої аудиторії. Групові подорожі є привабливим варіантом проведення вільного часу для багатьох людей, вони надають соціальну взаємодію, безпеку і можливість ділитися враженнями з іншими учасниками. Проте організація і координація групових подорожей можуть бути складними завданнями для організаторів і учасників.
На основі цієї потреби виникає необхідність у створенні програмної системи, яка об'єднала б у собі функціональність організації подорожей. Метою створення такої системи є спрощення процесу планування та координації групових подорожей, а також забезпечення безпеки та комфорту для учасників протягом усієї подорожі.

1.2	Business Opportunity (Бізнес-можливості)

На ринку мобільних додатків та вебсайтів для планування подорожей спостерігається значний попит. Система "TravelSync" конкурує з різноманітними програмами та сервісами, що надають послуги по організації подорожей. Зокрема, порівняно з "AllTrails", який спеціалізується на маршрутах для пішоходів та прогулянок, "TravelSync" вирізняється інтегрованим функціоналом для групових подорожей та відстеження учасників, що робить його більш зручним для організаторів і учасників.
"Viator", який спеціалізується на екскурсіях з гідом, не має інтегрованих інструментів для організації групових подорожей та моніторингу учасників, у той час, як "TravelSync" забезпечує ці функції, а також використовує IoT пристрої для покращення безпеки та комфорту під час подорожей.
У "Travello", соціальній мережі для мандрівників, також відсутній функціонал для організації та планування групових подорожей, що робить "TravelSync" більш привабливим варіантом для користувачів, які шукають інтегровані рішення.
У порівнянні з сайтами пошуку та бронювання подорожей, такими як "Kayak.com" та "Orbitz", які надають широкий спектр послуг, "TravelSync" видається більш важливою альтернативою для організаторів та учасників подорожей, оскільки він надає інтегрований функціонал для групових подорожей та відстеження учасників.
TravelSync пропонує розв’язання цієї проблеми, надаючи інтегрований функціонал для організації та координації групових подорожей. Крім того, використання інтернету речей (IoT) для відстеження учасників подорожей підвищує безпеку та комфорт, що робить TravelSync привабливим рішенням для організаторів та учасників подорожей.
TravelSync має кілька моделей монетизації, включаючи продаж спеціальних браслетів для відстеження учасників, отримання комісійних з платних екскурсій через платформу та розміщення реклами на своєму вебсайті або додатку. Такий підхід дозволяє TravelSync створювати прибуток та розвиватися на ринку мобільних додатків для подорожей.

2 VISION OF THE SOLUTION (КОНЦЕПЦІЯ РІШЕННЯ)
2.1 Vision Statement (Окреслення концепції)

TravelSync - інноваційний продукт для організації та координації групових поїздок, забезпечуючи безпеку, комфорт та зручність для учасників протягом усього подорожі. Наша мета - створити спільноту мандрівників, яка максимально використовує переваги сучасних технологій для полегшення планування та виконання поїздок, а також сприяти взаємодії та обміну досвідом між учасниками. Подорож - це не лише відпочинок, але й розширення горизонтів, знайомство з новими людьми та відкриття нових місць і культур. TravelSync призначений зробити цей процес ще більш приємним та безпечним для кожного учасника.
Система базується на використанні технологій, таких як GPS та пульсометри, для забезпечення точного відстеження місцезнаходження учасників та контролю їх фізичного стану під час поїздки. Окрім забезпечення зручних інструментів для планування маршрутів, система дозволяє взаємодіяти з іншими учасниками та отримувати необхідну підтримку у разі надзвичайних ситуацій.
Мета системи - досягнути максимального задоволення користувачів та забезпечити їм незабутні поїздки. TravelSync пропонує широкий спектр функцій для користувачів, спрощуючи групові поїздки та роблячи їх більш безпечними та зручними. Користувачі можуть створювати та редагувати свої маршрути, обирати цікаві місця та події для включення до плану поїздки, знаходити інші групи або створювати свої, запрошувати інших користувачів. За допомогою GPS-браслетів користувачі можуть відстежувати своє місцезнаходження під час поїздки та контролювати місцезнаходження інших учасників групи. Вони зможуть обмінюватися повідомленнями та інформацією з іншими учасниками групи, спілкуватися з організаторами поїздки та отримувати оновлення про події та заходи. У разі надзвичайних ситуацій користувачі можуть викликати служби екстреної допомоги та отримати допомогу за допомогою вбудованих функцій системи. Після завершення поїздки користувачі можуть ознайомитися зі своїми досягненнями, фотографіями та іншими даними, зібраними під час поїздки, для подальшого аналізу та обговорення з іншими учасниками.

2.2 	Major Functions (Головна функціональність)

MF-1: Можливість створення облікового запису користувача, включаючи введення особистої інформації та налаштування профілю. 
MF-2: Користувачі можуть створювати нові маршрути подорожей, обирати місця призначення, додавати визначні місця, події та інші деталі.
MF-3: Можливість пошуку та перегляду групових поїздок, вибору поїздок для приєднання або створення власних груп та запрошення інших користувачів.
MF-4: Користувачі можуть відстежувати своє місцезнаходження під час поїздки за допомогою GPS-браслетів та переглядати місцезнаходження інших учасників групи. 
MF-5: Можливість обміну повідомленнями та інформацією з іншими учасниками подорожей та організаторами. 
MF-6: Вбудовані функції екстреної допомоги та можливість виклику служб екстреної допомоги в разі потреби.

3 SCOPE AND LIMITATIONS (РАМКИ ТА ОБМЕЖЕННЯ)
3.1 Scope of Initial Release (Рамки первинного випуску)

Програмне забезпечення серверної частини системи:
 Серверна частина системи матиме розподілену архітектуру для ефективної обробки запитів користувачів та забезпечення безперервної роботи. Основними функціями серверної частини будуть:
-	Управління обліковими записами користувачів, включаючи реєстрацію, авторизацію та керування даними профілю;
-	Зберігання та керування даними про подорожі, включаючи створення, редагування та видалення маршрутів, а також обробку запитів про приєднання до групових подорожей;
-	Реалізація системи взаємодії з GPS-даними для відстеження місцеперебування учасників подорожей та надання екстрених служб у випадку потреби;
-	Забезпечення безпеки та конфіденційності даних користувачів шляхом шифрування та захисту від несанкціонованого доступу.
Програмне забезпечення пристроя IoT:
- Відстеження показників пульсу та координат користувача, передача даних іншим учасникам подорожі.
Веб застосунок:
Клієнтська частина системи буде надавати зручний та інтуїтивно зрозумілий інтерфейс для користувачів. Основні можливості клієнтської частини включатимуть реєстрацію та авторизацію користувачів для доступу до системи, перегляд, редагування та керування особистим профілем користувача, пошук та перегляд доступних подорожей, вибір та приєднання до них, інтерактивну карту з відображенням маршрутів та місцеперебування учасників, обмін повідомленнями та інформацією з іншими учасниками подорожей та організаторами.
Програмний застосунок для мобільного пристроя:
Мобільний додаток буде надавати зручний спосіб доступу до системи для користувачів, які перебувають у дорозі. Основні функції мобільного додатка включатимуть:
-	Зручний доступ до основних функцій системи через мобільний пристрій;
-	Відстеження місцеперебування та отримання сповіщень про стан подорожі;
-	Можливість виклику екстрених служб у випадку потреби;
-	Інтеграція з функціями камери та геолокації для зручності користувачів під час подорожей.

3.2 Scope of Subsequent Releases (Рамки наступних випусків)

Після початкового випуску передбачається розвиток та покращення системи шляхом впровадження нового функціоналу та поліпшення наявного. Очікувані напрямки розвитку включають:
-	Розширення можливостей аналізу даних та статистики для оптимізації процесів планування подорожей та взаємодії з користувачами;
-	Впровадження додаткових можливостей для персоналізації профілів користувачів та підвищення рівня безпеки та конфіденційності інформації;
-	Вдосконалення системи екстрених повідомлень та реагування на критичні ситуації для забезпечення безпеки учасників подорожей.

4 BUSINESS CONTEXT (БІЗНЕС-КОНТЕКСТ)
4.3	Operating Environment (Операційне середовище)

Робоче середовище проекту складається з наступних технологій для розробки своїх компонентів: 
База даних: MS SQL Server буде розгортатися на сервері з операційною системою Windows Server або Windows 10, забезпечуючи стабільну та швидку роботу бази даних.
Серверна частина: Використання  C# і .NET 8 у серверної частини системи "TravelSync" забезпечить ефективну обробку запитів користувачів та безперервну роботу, ASP.NET Core 8.0 використовуватиметься для API, Для взаємодії з базою даних використовуватиметься Entity Framework 8.0.
Вебдодаток: Angular Framework версії 17 із використанням мови програмування TypeScript.
Мобільний додаток: Використання Java Android (Android SDK, minimum SDK – API 24: Android 7.0) для мобільного додатка забезпечить доступ користувачів до системи з різних пристроїв з операційною системою Android. 
IoT: Використання платформи Arduino для побудови смартпристроїв гарантує їх взаємодію з системою та збір та передачу даних.
Для правильного функціонування системи треба врахувати різницю в часових поясах для забезпечення доступу до системи, розробити механізми обробки та передачі даних, які генеруються в різних місцях, забезпечити надійність та постійний доступ до системи. Необхідно взяти до уваги захист особистих даних користувачів через контроль доступу та безпеку системи.
